PrOH Tool© User Instructions


A. Filling in the MS Excel SIPOC table with process data
  1. Once downloaded make column widths of the SIPOC table suitable for your own use
  2. The top row of the SIPOC table contains the heading of the SIPOC table and does not become part of the PrOH Model
  3. The second row down forms the core process statement of the PrOH Model
  4. Add further lines of the SIPOC table
  5. Decision bubbles should be written as seen below (Figure 1) where words in brackets become sub-nodes and the blank line means that blank cells are the same as the cells above and so do not need to be repeated
  6. All outputs should also be shown as inputs so that there are no loose ends in the logic of the SIPOC table.

Once completed it is a good idea to use ‘save as’ to save your file as a MS Excel file to preserve your personal formatting preferences. A simple example about making a car is given in Figure 1. Once ready to upload as a .csv file do so but note that this will remove any personal formatting preferences you may have made to the SIPOC Table.


Figure 1: Example of filled in SIPOC table with personal formatting preferences

PrOH Tool performs a verification check on the filled in SIPOPC Table .csv file and if it is not filled in correctly PrOH Tool© will not allow the MS PowerPoint with the draft PrOH Model to be downloaded. Further changes can be made to the data by returning to your MS Excel version (.xlsx) of the SIPOC file and repeating activities from Steps 2 onwards. It is not necessary to start with a new empty template. Remember to press the ‘New’ button on PrOH Tool© before re-uploading.


B. Completing the PrOH Model drawing from the downloaded draft PowerPoint Model
  1. Open the downloaded MS PowerPoint file. Figure 2 is an example showing elements from Figure 1 spread across five different slides.
  2. Cut and paste elements onto a single slide removing any duplicated elements
  3. Create a title which needs to be the same as the central part of the core process statement
  4. Add connecting arrows – thicker ones for the core process statement and thinner ones for the non-core process arrow links and place verb phrases on the arrow links
  5. Adapt text to make a meaningful model that meets with PrOH Model rules and guidelines
  6. Make the model look appealing. Figure 3 is an example created from elements in Figure 2.
